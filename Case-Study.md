# Blockchain Staking DApp Scrum Case Study

## Background

A blockchain startup required a staking platform allowing users to stake tokens and earn rewards securely.

---

## Problem

The project required an iterative development process to deliver features incrementally while maintaining flexibility for stakeholder feedback.

---

## Scrum Approach

The project was managed using Scrum practices.

Activities included:

- Product Backlog Refinement
- Sprint Planning
- Daily Scrum
- Sprint Review
- Sprint Retrospective

---

## Sprint 1

Focus:

Core staking functionality.

Completed:

- Create Staking Smart Contract
- Implement Reward Calculation
- Create Wallet Connection

---

## Sprint 2

Focus:

Deployment and usability improvements.

Completed:

- Build Staking Dashboard
- Deploy to Sepolia Testnet
- Perform End-to-End Testing

---

## Outcome

The project successfully delivered a staking application using Scrum principles and Agile practices.

---

## Lessons Learned

- Early stakeholder feedback improves delivery quality.
- Smaller sprint increments improve predictability.
- Continuous testing reduces deployment risks.
